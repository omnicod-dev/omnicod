# Copywriting Frameworks — Satış Metni YAZIM REHBERİ

**Odağ:** AIDA, PAS, FAB formülleri ile dönüşüm odaklı satış metinleri üretmek.

---

## 1. AIDA Framework (En Yaygın)

```
AIDA = Attention (Dikkat) → Interest (İlgi) → Desire (İstek) → Action (Eylem)

Adımlar:
├── ATTENTION (Dikkat Çek): Güçlü başlık veya açılış cümlesi
│   ├── Problem cümlesi: "X yapmak için yıllardır çabalıyor musunuz?"
│   ├── Şok edici istatistik: "Mevcut e-postalarınızın %85'i okunmuyor."
│   ├── Olağandışı iddia: "Bu yöntemle 3 ayda 10 yıl deneyim kazanacaksınız."
│   └── Hemen dikkat dağıtmak: "[İsim], sizi tanıyorum çünkü..."
│
├── INTEREST (İlgi Uyandır): Argümanları geliştir
│   ├── Ürün/hizmetin ne olduğunu kısaca tanıt
│   ├── Hedef kitleye özel faydalar listele (özellik değil fayda!)
│   └── "Şu anda X sorunu mu yaşıyorsunuz?" gibi soru sor
│
├── DESIRE (İstek Yarat): Faydaları duygusal hale getir
│   ├── Hayal kurdur: "Bir hafta sonra X hissedeceksiniz."
│   ├── Sosyal kanıt ekle: "10.000+ kişi zaten kullanıyor."
│   ├── Scarcity/urgency: "Sadece 23 kişilik kontenjan kaldı."
│   └── Risk azaltma: "30 gün içinde memnun değilseniz para iade."
│
└── ACTION (Eyleme Geçir): Net CTA
    ├── CTA buton metni: "Şimdi Başla" → "Hemen Erişim Al"
    ├── Birinci tekil şahıs: "Ben" yerine "sen" kullan
    └── Aciliyet unsuru: "Bu fırsat yarın sona erecek."
```

---

## 2. PAS Framework (Problem-Agitate-Solution)

```
PAS = Problem → Agitate → Solution

Uygulama:
├── PROBLEM: Hedef kitlenin yaşadığı sorunu tanımla
│   ├── "X yapmak istiyorsunuz ama yapamıyorsunuz."
│   ├── "Şu anki çözümleriniz yeterli değil."
│   └── Müşterinin kendi ağzından sorun
│
├── AGITATE (Sorunu Şişir): Acıyı derinleştir
│   ├── Sorunun sonuçlarını anlat: "Bu, şirketinize ayda X TL kayba mal oluyor."
│   ├── Başka çözüm deneyip başarısız olanları anlat
│   ├── "Farkında olmadan..." gibi duygu yüklü ifadeler
│   └── "Her gece bunu düşünerek uyuyamıyorsunuz." gibi empati
│
└── SOLUTION (Çözüm Sun): Ürünle sorunu çöz
    ├── "İşte size X ürünümüz."
    ├── Nasıl çalıştığını adım adım anlat
    ├── Somut sonuçları listele
    └── CTA ile bitir
```

---

## 3. FAB Framework (Feature-Advantage-Benefit)

```
FAB = Özellik → Avantaj → Fayda Dönüşümü

Prensip: Her özelliği faydaya çevir.

Örnek dönüşüm:
├── ❌ "5000 mAh pil kapasitesi" (özellik — teknik detay)
├── ✅ "Tam gün çalışsanız bile piliniz bitmeyecek" (fayda — duygusal)
└── ✅ "Raporunuzu sunarken şarj adaptörü aramak zorunda kalmayacaksınız." (sonuç)

FAB Analiz Şablonu:
Ürün Özelliği: [teknik özellik]
Bu Ne İşe Yarar: [nasıl çalışır]
Müşteri İçin Faydası: [hayatında ne değişir]
```

## 4. Başlık Formülleri (Headline Templates)

```
Formula kategorileri:
├── SAYI İÇEREN:
│   ├── "X Adımda [Hedef Sonuç]"
│   ├── "[Sayı] Kişinin %X'i [Yapıyor]"
│   └── "[Sayı] Tane [Beklenmedik Fayda]"
│
├── SORU TABANLI:
│   ├── "[Endişe] mu? İşte kesin çözüm."
│   ├── "[Hayal Kırıklığı] mı yaşıyorsunuz? [Alternatif] deneyin."
│   └── "[İddia] — Gerçek mi?"
│
├── KORKU/ÖFKE:
│   ├── "[Yapmazsan] pişman olacağın X şey."
│   └── "X Yapanların %90'ı [Yanlış Yapıyor]"
│
└── SOSYAL İSPAT:
    ├── "[Uzman/Kişi sayısı] Kişi Zaten [Yapıyor]"
    └── "Neden [X] Bu Kadar Çok [Tercih Ediliyor]?"
```

## 5. Body Copy Yapıları

```
Satış Mektubu Formatı:
├── Üst Kısım (Hero):
│   ├── Başlık (H1)
│   └── Alt başlık (hedef kitle + sonuç vaadi)
│
├── Body:
│   ├── Problem tanımı (1-2 paragraf)
│   ├── Çözüm tanıtımı (2-3 paragraf)
│   ├── Özellik listesi (madde işaretli)
│   ├── Sosyal kanıt (müşteri yorumları, logolar)
│   └── Fiyat/nakit akışı (varsa)
│
└── Alt Kısım:
    ├── CTA tekrarı
    ├── Güvence tekrarı (para iade vb.)
    └── İletişim bilgileri
```

## 6. CTA (Call to Action) Best Practices

```
CTA Türleri:
├── DIRECT CTA: "Şimdi Satın Al", "Hemen Başla"
├── SOFT CTA: "Daha Fazla Bilgi Al", "Broşür İndir"
├── SCARCITY CTA: "Sınırlı Kontenjan", "Fırsat Sona Eriyor"
└── REASSURANCE CTA: "Ücretsiz Deneyin", "Risksiz Başla"

CTA Yazım Kuralları:
├── Birinci şahıs kullanma: "Tıkla" değil, "Al"
├── Aciliyet yarat: "Şimdi" veya "Bugün" ekle
├── Faydayı vurgula: " Ücretsiz Başla" > "Kaydol"
├── Buton rengi ve konumu kritik
└── Mobilde parmakla tıklanabilir boyut (min 44px)
```

## 7. Duygu Tetikleyicileri (Emotional Triggers)

```
Priming Kategorileri:
├── TEMEL DUYGULAR (en güçlü):
│   ├── Korku: "Yapmazsan kaybedeceksin."
│   ├── Açgözlülük: "Bu fırsatı kaçırma."
│   ├── Merak: "X sırrını öğrenmek ister misin?"
│   └── Öfke: "X şirketi sizi kandırıyor."
│
├── SOSYAL DUYGULAR:
│   ├── Ait olma: "X kişilik topluluğa katıl."
│   ├── Prestij: "X kullananlardan biri ol."
│   └── Rekabet: "Rakiplerinizden önde olun."
│
└── RASYONEL DUYGULAR:
    ├── Mantık paketlemesi: Verilerle destekle
    ├── Otorite: "Uzmanlar diyor ki..."
    └── Sosyal ispat: "10.000+ kişi tercih etti."
```

## 8. Sosyal Kanıt Türleri

```
Sosyal Kanıt Hiyerarşisi (güvenilirlik sırası):
├── 1. Video testimonial (en güçlü)
├── 2. Detaylı yazılı testimonial
├── 3. Müşteri logosu/branded logo
├── 4. Medya alıntısı (X'te yayınlandı)
├── 5. Sayısal veri: "X kişi kullandı"
└── 6. Uzman onayı/endorsement

Ekran görüntüsü kullanımı:
├── Gerçek tarihli olmalı
├── Müşteri adı soyadı görünür olmalı
├── Profil fotoğrafı ekle
└── Ürün adı açıkça yazılı olmalı
```

## Key Patterns

1. **Her zaman AIDA veya PAS kullan** — Plansız yazılmış satış metni düşük dönüşüm sağlar.
2. **Özellik değil fayda sat** — Müşteri ne alacağını değil, hayatının nasıl değişeceğini düşünür.
3. **Tek CTA kuralı** — Bir sayfada/mesajda sadece bir net eylem iste.
4. **Sosyal kanıt zorunlu** — Güven inşası için en az bir tür sosyal kanıt ekle.
5. **Aciliyet + scarcity** — Dönüşümü %30-50 artırır.

---

## Anti-Patterns

```
❌ "Kaliteli, profesyonel, güvenilir hizmet" gibi soyut kelimeler
✅ Somut, ölçülebilir sonuçlar: "2 haftada %40 daha fazla satış"

❌ Uzun paragraflar (10+ cümle) — okunmaz
✅ Kısa paragraflar, madde işaretleri, bold anahtar kelimeler

❌ Birden fazla CTA — kullanıcı karar veremez
✅ Tek net CTA: "Şimdi Başla" veya "Ücretsiz Dene"

❌ "Tıklayın", "Gönderin" gibi zayıf CTA'lar
✅ "Erişim Al", "Önce Siz Deneyin", "Yerini Ayır"

❌ Genel vaatler: "Hayatınız değişecek!"
✅ Spesifik vaatler: "60 günde İngilizce B2 seviyesine ulaşacaksınız."

❌ Yalan veya abartılmış iddialar
✅ Kanıtlanabilir, desteklenebilir veriler
```

## Quick Reference

| Framework | Kullanım Alanı | Odak Noktası |
|---|---|---|
| AIDA | Genel satış metinleri, landing page | Dikkat → Eylem zinciri |
| PAS | Duygusal satış, problem çözümü | Sorun şişirme |
| FAB | Teknik ürün tanıtımı | Özellik → Fayda dönüşümü |
| Problem-Solution | Eğitim, yazılım tanıtımı | Issue → Fix modeli |
| StoryBrand | Marka hikayesi | Müşteri kahraman modeli |

| Başlık Türü | Örnek | Etki |
|---|---|---|
| Sayısal | "7 Adımda X" | Somut, adım adım vaat |
| Soru | "Neden X?" | Merak uyandırır |
| Korku | "Yapmazsan..." | Aciliyet yaratır |
| Sosyal | "X kişi tercih etti" | Güven sağlar |
| Sonuç odaklı | "X yapmanın tek yolu" | Net beklenti |

| CTA Türü | Örnek | Strateji |
|---|---|---|
| Direct | "Şimdi Satın Al" | Hazır müşteri |
| Soft | "Daha Fazla Bilgi" | Bilgi arayan |
| Scarcity | "Sınırlı Kontenjan" | Kararsız |
| Reassurance | "Ücretsiz Dene" | Riskten kaçınan |
